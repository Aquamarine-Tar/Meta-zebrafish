# 鱼体游动 FSI 显式弱耦合仿真项目说明

本文档面向课程汇报，重点解释当前项目解决的力学问题、采用的仿真方法和主要参数。项目总体思路不是“基于某个代码框架的优化”，而是针对鱼体游动这一典型流固耦合问题，构建一个显式弱耦合仿真系统：

- 固体侧：将鱼体看作可大变形的软体结构，基于 Softcon 的仿真思路进行求解。
- 驱动侧：使用沿身体传播的肌肉激活波产生鱼体摆动。
- 流体侧：使用 GPU 粒子流体，主体为 WCSPH，近壁压力使用 VAP band 投影增强。
- 耦合侧：每个时间步先由鱼体表面作为运动边界驱动流体，再把流体压力/近壁力转换为 FEM 顶点外力，推进鱼体下一步运动。

一句话概括：这是一个“软体鱼结构动力学 + 粒子流体动力学 + 显式 FSI 力交换”的仿真实验平台。

## 1. 力学问题：鱼体游动的流固耦合

鱼体游动本质上是流固耦合问题。鱼体内部肌肉产生周期性收缩，使柔性身体发生侧向波动；身体表面推动周围水体，水体反过来在鱼体表面产生压力和剪切作用。这些流体力进一步影响鱼体变形、姿态和推进效果。

从力学上看，系统包含两类未知量：

1. 固体未知量：鱼体材料点的位置、速度、变形、应力和肌肉主动收缩。
2. 流体未知量：水体粒子的位置、速度、密度、压力和近壁投影压力。

理想的连续问题可以写成：

```text
固体区域 Ω_s(t):    ρ_s x¨ = div(P) + f_muscle + f_fluid
流体区域 Ω_f(t):    ρ_f Dv/Dt = -∇p + μ∇²v + f_body
不可穿透边界 Γ(t):  v_fluid · n = v_solid · n
界面力平衡:          traction_solid + traction_fluid = 0
```

本项目不直接求解全连续强耦合形式，而是采用离散显式弱耦合：

```text
已知 t_n 时刻鱼体形状和速度
        |
        v
把鱼体表面上传为流体运动边界
        |
        v
流体子步进，计算近壁压力和水动力
        |
        v
将流体力分配到鱼体 FEM 顶点
        |
        v
固体 FEM 推进到 t_{n+1}
```

这种做法牺牲了一部分强耦合精度，但优点是结构清晰、模块独立、调试容易，适合课程项目中展示完整 FSI 计算链路。

## 2. 总体数值框架

当前仿真的一个主时间步可以表示为：

```text
1. 固体侧已有 x_s^n, v_s^n
2. 提取鱼体表面三角网格 Γ_s^n
3. 将 Γ_s^n 和表面速度 v_s^n 作为流体边界条件
4. 流体在一个固体步内执行 N_sub 个 SPH/VAP 子步
5. 流体得到边界力 F_fluid^n
6. 对子步力做平均、截断和表面补齐
7. 将 F_fluid^n 作为外力施加到 FEM 顶点
8. 固体侧用 Projective Dynamics 更新 x_s^{n+1}, v_s^{n+1}
```

项目中的主循环对应 `Environment::Step()`，但汇报时更建议把它讲成如下耦合格式：

```text
Fluid boundary update:
    Γ_s^n = surface(x_s^n)
    u_Γ^n = v_s^n

Fluid solve:
    F_Γ^n = WCSPH/VAP(Γ_s^n, u_Γ^n, Δt_s)

Solid solve:
    M(x_s^{n+1} - x_s^n) / Δt_s² = internal_PD + muscle + F_Γ^n
```

这里的关键点是：流体使用的是固体当前步的几何，固体使用的是本步流体计算出的外力，二者不在同一个时间步内反复迭代，所以称为显式弱耦合。

## 3. 固体侧：软体鱼 FEM 与 SoftCon/Projective Dynamics

### 3.1 鱼体离散模型

鱼体被离散为四面体网格。当前默认鱼体文件为 `data/fish_body.mesh`：


| 项目    | 数量           |
| ----- | ------------ |
| 顶点    | 7950         |
| 四面体   | 24676        |
| 显式三角面 | 0，由四面体边界自动提取 |


`data/fish.meta` 描述鱼体模型装配关系：

```text
0.004
fish_body.mesh
zf_nerve_cord.obj
fish.sampling
0 4072
6466 5760 5781
```

含义是：缩放鱼体网格，读取四面体身体，读取肌肉/神经线几何，读取采样点，并指定头尾/局部坐标参考点。

### 3.2 被动单元、主动肌肉单元与神经索定义

固体侧可以分成两类力学单元：

1. 被动单元：四面体 FEM 单元，描述鱼体组织本身的弹性、体积保持和大变形响应。
2. 主动单元：沿神经索/肌肉路径布置的肌肉段，描述由神经信号或预设波形驱动的主动收缩。

被动单元来自 `fish_body.mesh` 的四面体。每个四面体只承担材料本体的力学行为：拉伸、压缩、剪切和旋转后的恢复。也就是说，即使没有肌肉激活，鱼体仍然是一个具有质量、刚度和阻尼的软体结构，会对外力产生弹性变形。

主动肌肉单元不是在网格中额外生成实体单元，而是由 `zf_nerve_cord.obj` 中的折线几何定义。这个 OBJ 文件包含多个对象 `o ...` 和线段 `l i j`，可以理解为沿鱼体长度方向布置的神经索/肌肉纤维路径。程序读取每条折线后，将其按弧长采样成若干肌肉段；当前每条肌肉最多采样为 24 段。

力学映射过程如下：

```text
神经索/肌肉折线
        |
        v
按弧长采样为肌肉段 start-end
        |
        v
寻找被该段穿过或靠近的四面体
        |
        v
在这些四面体上添加 LinearMuscleConstraint
        |
        v
激活值 a(t, s) 改变该方向上的主动收缩约束
```

因此，“神经索”在当前仿真中主要承担几何和驱动路径的作用：它告诉系统主动收缩波沿哪些空间路径传播，以及这些主动收缩应该作用到鱼体内部哪些四面体上。真正承受变形和传递内力的仍然是四面体 FEM 网格。

肌肉段和被动四面体之间通过距离权重耦合。若肌肉线段穿过某个四面体，或距离该四面体中心足够近，就在该四面体上添加沿肌肉方向的主动约束；距离越近权重越大。当前权重形式为：

```text
w = exp(-|d| / 0.005)
```

这种定义方式的好处是，肌肉不需要和四面体网格一一对齐。只要给出一组沿身体分布的神经索/肌肉折线，就可以自动把主动收缩映射到附近的被动组织单元上。

### 3.3 固体动力学方程

离散后的鱼体可以写成标准有限元动力学形式：

```text
M x¨ = f_internal(x) + f_muscle(x, a) + f_fluid + f_damping
```

其中：

- `M` 是质量矩阵。
- `x` 是所有 FEM 顶点位置。
- `f_internal` 来自弹性材料变形。
- `f_muscle` 来自主动肌肉约束。
- `f_fluid` 是流体侧回传的外力。
- `f_damping` 是数值阻尼。

当前固体侧采用 SoftCon 类 Projective Dynamics 思路，不是直接显式积分弹性力，而是把材料约束、肌肉约束写成可投影约束，并通过局部-全局迭代稳定求解大变形。

### 3.4 Corotated FEM：处理大旋转软体变形

鱼体在游动时会发生较大整体旋转和局部弯曲。如果使用线性小变形模型，旋转会被错误地解释成应变，导致非物理刚化。因此固体侧采用 Corotated FEM 约束。

每个四面体静止构型有边矩阵：

```text
Dm = [X1 - X0, X2 - X0, X3 - X0]
```

变形后有：

```text
Ds = [x1 - x0, x2 - x0, x3 - x0]
F  = Ds Dm^{-1}
```

其中 `F` 是变形梯度。Corotated 思路将 `F` 分解为旋转和拉伸：

```text
F ≈ R S
```

弹性能主要惩罚去掉刚体旋转后的形变：

```text
E ≈ μ ||F - R||² + λ tr(RᵀF - I)²
```

这样鱼体整体转动不会产生虚假的弹性应力，而弯曲、压缩、拉伸会产生恢复力。

当前主要材料参数：


| 参数           | 当前值   | 含义            |
| ------------ | ----- | ------------- |
| 默认杨氏模量       | `2E5` | 鱼体大部分区域刚度     |
| label=1 杨氏模量 | `4E5` | 特定材料区域更硬      |
| 泊松比          | `0.4` | 接近软组织的近不可压缩趋势 |
| 身体质量         | `10`  | 均匀分配到顶点       |


### 3.5 Projective Dynamics：局部投影 + 全局求解

Projective Dynamics 的核心是把复杂的非线性弹性问题转化为两个交替步骤：

1. 局部步：每个约束独立计算“最接近当前状态的合法形变”。
2. 全局步：解一个稀疏线性系统，使所有顶点尽量满足这些局部投影。

从能量角度，可以理解为最小化：

```text
E(x) = inertial_term + Σ_i w_i ||A_i x - p_i||²
```

其中：

- `inertial_term` 保证时间推进符合惯性。
- `A_i x` 是当前顶点组合形成的约束量。
- `p_i` 是局部投影得到的目标形状。
- `w_i` 是约束刚度。

每个时间步先根据外力预测惯性位置：

```text
q_n = x_n + Δt v_n + Δt² M^{-1} f_ext
```

然后求解：

```text
((1/Δt²) M + L) x_{n+1} = (1/Δt²) M q_n + J d
```

其中：

- `L` 是所有约束贡献的系统矩阵。
- `J d` 是局部投影后的右端项。
- 矩阵在拓扑和时间步固定时可以预分解，因此求解效率较高。

速度更新：

```text
v_{n+1} = (x_{n+1} - x_n) / Δt
v_{n+1} ← damping · v_{n+1}
```

当前固体侧主参数：


| 参数      | 当前值       |
| ------- | --------- |
| 固体仿真频率  | `960 Hz`  |
| 固体时间步   | `1/960 s` |
| PD 最大迭代 | `50`      |
| 速度阻尼    | `0.9999`  |


### 3.6 肌肉驱动：沿身体传播的主动收缩波

鱼体游动的主动输入来自上面定义的肌肉单元。力学上，它模拟的是身体两侧肌肉交替收缩，产生从头到尾传播的弯曲波。神经索/肌肉折线给出空间路径，激活函数给出每个路径位置和时间上的收缩强度。

当前测试中常用正弦激活：

```text
a(s, t) = offset + A sin(2π f t + sign · 2π k s + φ_side)
```

其中：

- `s` 是沿鱼体长度方向的归一化位置。
- `A` 是激活振幅。
- `f` 是时间频率。
- `k` 是身体上的空间波数。
- `φ_side` 使左右侧肌肉相差 `π`，形成交替收缩。

默认参数：


| 参数             | 值        |
| -------------- | -------- |
| amplitude      | `0.25`   |
| offset         | `0.25`   |
| frequency      | `1.0 Hz` |
| spatial cycles | `1.0`    |
| 左右相位差          | `π`      |
| 激活范围           | `[0, 1]` |


肌肉不是简单地给某些顶点施加外力，而是作为内部主动约束影响四面体。每条神经索/肌肉线被采样成多个肌肉段，每段找到附近或穿过的四面体，并添加沿肌肉方向的 `LinearMuscleConstraint`。这样肌肉收缩会自然转化为身体内部应变，再通过被动四面体材料传播到整个身体，而不是变成直接作用在表面的外力。

## 4. 流体侧：WCSPH 粒子流体

### 4.1 为什么使用 SPH

鱼体表面在游动中不断变形，传统网格流体需要处理复杂的移动边界和网格更新。SPH 是无网格拉格朗日粒子法，流体由粒子携带质量、速度、密度和压力，适合处理移动边界和大变形界面。

和传统网格方法相比，可以简单概括为：


| 方面       | 传统网格方法                      | SPH 粒子方法                                  |
| -------- | --------------------------- | ----------------------------------------- |
| 基本描述     | 在固定或变形网格上存储速度、压力等场量         | 粒子随流体运动，每个粒子携带质量、速度、密度、压力                 |
| 移动边界     | 需要浸入边界、ALE 动网格或重剖分来处理鱼体运动   | 鱼体表面可直接作为运动边界影响附近粒子                       |
| 大变形/自由表面 | 需要额外界面追踪或重构，如 level set、VOF | 粒子天然跟随界面运动，拓扑变化处理更直接                      |
| 压力精度     | 网格上求解压力泊松方程，压力场通常更平滑准确      | 压力由邻域粒子估计，近壁和低分辨率时噪声更明显                   |
| 守恒性      | 有成熟有限体积/有限元格式，守恒性和收敛理论更完善   | 质量守恒自然，但压力和边界处理依赖核函数与邻域质量                 |
| 边界条件     | 固壁无穿透/无滑移可以较严格施加，但移动几何复杂    | 移动边界简单，但需要 ghost/boundary particles 或投影修正 |
| 计算特点     | 稀疏线性系统和网格数据结构，适合高精度 CFD     | 邻域搜索和粒子相互作用高度并行，适合 GPU                    |
| 本项目取舍    | 边界精度更强，但实现复杂度高              | 更适合快速实现鱼体大变形 FSI，因此采用 WCSPH + VAP 修正近壁压力  |


所以，本项目选择 SPH 的主要原因不是它在所有方面都优于网格法，而是它更适合当前任务的工程重点：鱼体边界持续变形、需要和 FEM 表面快速交换力、希望在 GPU 上并行运行，并且可以通过 VAP band 专门补强 SPH 在近壁压力上的短板。

在 SPH 中，任意连续场 `A(x)` 由邻域粒子核插值：

```text
A(x_i) ≈ Σ_j m_j A_j / ρ_j · W(|x_i - x_j|, h)
```

其中：

- `W` 是光滑核函数。
- `h` 是核半径。
- 邻居通常只搜索 `r < h` 的粒子。

当前项目用均匀网格加速邻域搜索。每个粒子只需检查自己所在格子附近的格子，避免 O(N²) 全局搜索。

### 4.2 WCSPH：弱可压缩 SPH

当前流体主体是 WCSPH，即弱可压缩 SPH。它不严格解不可压约束，而是允许密度有小幅波动，再通过状态方程把密度变化转成压力。

密度估计：

```text
ρ_i = Σ_j m_j W_ij
```

代码中还做了下限处理：

```text
ρ_i = max(ρ_i, ρ0)
```

压力通过 Tait 状态方程计算：

```text
p_i = ρ0 c_s² / γ · [ (ρ_i / ρ0)^γ - 1 ]
γ = 7
p_i = max(p_i, 0)
```

这里 `c_s` 是人工声速。声速越高，流体越接近不可压，但时间步稳定性要求更严格；声速越低，计算更稳定但密度波动更大。

当前主要流体参数：


| 参数        | 当前值          | 含义      |
| --------- | ------------ | ------- |
| 粒子间距 `dx` | `0.011 m`    | 流体分辨率   |
| 核半径 `h`   | `2 dx`       | SPH 支撑域 |
| 参考密度 `ρ0` | `1000 kg/m³` | 水密度     |
| 人工粘性      | `0.01`       | 数值粘性/耗散 |
| 声速 `c_s`  | `20 m/s`     | 控制可压缩性  |
| 重力        | `0`          | 当前不考虑重力 |


### 4.3 SPH 力

压力力使用 Spiky 核梯度，形式为：

```text
f_i^p = -Σ_j m_j (p_i + p_j) / (2ρ_j) ∇W_ij
```

粘性力使用粘性核拉普拉斯：

```text
f_i^v = Σ_j m_j μ (v_j - v_i) / ρ_j ∇²W_ij
```

从物理含义看：

- 压力项抵抗粒子局部压缩，推动高密度区域向低密度区域释放。
- 粘性项耗散相邻粒子速度差，抑制数值噪声和局部穿插。

在当前实现中，远离鱼体的 bulk 区域主要由 WCSPH 压力和粘性维持流体稳定。

## 5. 近壁流体：VAP band 压力投影

### 5.1 WCSPH 近壁问题

鱼体 FSI 的关键不是远场水体，而是鱼体表面的近壁压力分布。单纯 WCSPH 在近壁会遇到几个问题：

1. 鱼体表面 FEM 顶点数量远多于 SPH 粒子，压力覆盖可能很稀疏。
2. 粒子与三角面边界之间可能出现 gap，导致部分表面没有有效水压力。
3. ghost 边界过强时会主导局部线性系统，过弱时又难以保证不可穿透。
4. WCSPH 的压力来自密度波动，近壁处邻域不完整，压力估计容易噪声大。

因此当前项目在鱼体附近引入 VAP band，即只对靠近鱼体表面的一层流体粒子做压力投影。

### 5.2 VAP 的总体目标和线性系统

VAP 可以理解为近壁区域的压力投影修正。它的目标不是替代整个 WCSPH 流体，而是在鱼体表面附近构造一个局部压力投影，使近壁粒子更接近不可压，并更好地响应鱼体表面运动边界。

概念上，投影步骤类似不可压流体中的压力泊松方程：

```text
∇²p = ρ / Δt · ∇ · u*
```

在当前实现中，这个方程被写成近壁粒子邻域上的线性系统：

```text
A p = b
```

其中：

- `p` 是 VAP band 内流体粒子的投影压力。
- `b` 是由速度散度、边界运动和密度补偿构造的源项。
- `A` 是由粒子邻域、核函数权重和对角补偿构造的压力投影算子。
- 求解后，压力既用于修正近壁粒子速度，也用于积分鱼体表面水压力。

### 5.3 特别设计一：band 内外混合

当前方法不是全域都使用 VAP，而是采用 band 内外混合策略：

- band 内：靠近鱼体表面的粒子使用 VAP 压力投影，重点处理边界不可穿透和近壁压力覆盖。
- band 外：远离鱼体的 bulk 流体继续使用 WCSPH 压力和粘性，维持整体流场和粒子分布。
- band 内仍保留粘性作用，用于速度耗散和数值稳定。

这个设计的动机是：鱼体 FSI 最敏感的是近壁压力，没必要在整个流体域上求解更昂贵的投影系统。只在近壁 band 内做 VAP，可以把计算量集中在真正影响鱼体受力的区域。

流程上可以理解为：

```text
WCSPH 计算全域密度/基础压力
        |
        v
标记鱼体附近的 band 粒子
        |
        v
band 外继续使用 WCSPH
band 内由 VAP 投影压力主导
        |
        v
用 VAP 压力积分鱼体表面水动力
```

### 5.4 特别设计二：ghost 边界粒子

ghost 粒子代表鱼体表面的运动边界。它们的位置和速度不是流体求出来的，而是由固体 FEM 表面给定：

```text
x_ghost = x_surface
v_ghost = v_surface
```

在一个固体步内的多个流体子步中，也可以用表面速度对 ghost 位置做近似推进：

```text
x_ghost(t + δt) ≈ x_surface(t) + v_surface · δt
```

力学上，ghost 粒子用于表达边界不可穿透条件：

```text
u_fluid · n ≈ u_solid · n
```

但是 ghost 粒子不作为压力自由度。它们只参与邻域计算、散度源项和速度惩罚，相当于告诉近壁流体“这里有一个正在运动的固体边界”。当前实现还使用 `ghost_div_scale` 和 `ghost_vel_scale` 减弱 ghost 影响，这是因为鱼体表面顶点密度通常高于 SPH 粒子密度；若 ghost 过强，近壁压力会被边界点主导，反而导致不稳定。

### 5.5 VAP 子步完整流程

每个流体子步中，VAP 的计算流程可以概括为：

```text
1. 标记 near-wall band 粒子
2. 构造 merged 粒子集合：band fluid + kinematic ghost
3. 在 merged 集合上建立邻居表
4. 计算 alpha 归一化系数
5. 计算压力矩阵对角项 aii，并检测 surface 粒子
6. 计算右端项 b，即 divergence source
7. 用共轭梯度求解 A p = b
8. 用压力 p 修正 band 粒子速度
9. 将压力 p 写回原流体粒子
10. 用 VAP 压力对鱼体表面做水压力积分
```

下面按这些步骤说明使用的主要公式。

#### 5.5.1 Step 1：标记 VAP band 粒子

对每个流体粒子 `x_i`，搜索鱼体表面三角形上的最近点 `q`。如果：

```text
||x_i - q|| <= band_h
```

则该粒子进入 VAP band。当前：

```text
band_h = h · VAP_BAND_H_MULT
VAP_BAND_H_MULT = 2.5
```

所以只有距离鱼体表面约 `2.5h` 以内的流体粒子参与近壁投影。这样把计算集中在 FSI 最关键的近壁区域，而不是全域求解压力投影。

#### 5.5.2 Step 2：构造 merged 粒子集合

VAP 不直接在所有 SPH 粒子上求解，而是构造：

```text
merged = band fluid particles + kinematic ghost particles
```

两类粒子的作用不同：

```text
fluid particle:  压力自由度，需要求解 p
ghost particle:  运动学边界，不求解 p，只提供边界位置和速度
```

这样做可以把运动边界并入粒子邻域计算，同时避免为固体表面点引入额外压力未知量。

#### 5.5.3 Step 3：建立邻居表

VAP 在 merged 粒子集合上建立邻居表。对每个 band fluid 粒子，邻居包括：

- 近邻 fluid 粒子。
- 少量最近 ghost 粒子。

当前限制：

```text
MAX_VAP_NEIGHBORS = 128
VAP_GHOST_NEIGHBOR_RESERVE = 8
```

保留 ghost 邻居配额的原因是：鱼体表面 ghost 点可能非常密，如果不限制数量，压力系统容易被 ghost 主导；而如果完全不用 ghost，又无法表达运动边界。

#### 5.5.4 Step 4：计算 alpha 归一化系数

对每个 VAP fluid 粒子，先计算邻域核权重和：

```text
α_i = Σ_j W(||x_i - x_j||, h)
```

这里使用 Poly6 核：

```text
W(r, h) = 315 / (64πh⁹) · (h² - r²)³,  r < h
```

并做数值下限：

```text
α_i = max(α_i, 1e-6)
```

随后记录全局 `alpha_max`，再修正：

```text
α_i = max(α_i, alpha_max)
```

这样做是为了避免近壁邻域缺失时 `α_i` 过小，导致后续 `1/α_i` 权重异常放大。

#### 5.5.5 Step 5：构造对角项 aii 和 surface 检测

先定义：

```text
wrr(r, h) = W(r, h) / (r² + ε)
```

对每个 fluid 粒子 `i`：

```text
inv_a = 1 / α_i
w_ij = inv_a · wrr(||x_i - x_j||, h)
```

如果邻居 `j` 是 fluid：

```text
aii_fluid[i] += w_ij
aii_total[i] += w_ij
aii_fluid[j] += w_ij
aii_total[j] += w_ij
```

如果邻居 `j` 是 ghost：

```text
aii_total[i] += VAP_GHOST_DIAG_COEFF · w_ij
VAP_GHOST_DIAG_COEFF = 1.0
```

随后检测 near-wall 或邻域不足的 surface 粒子，并对 `aii` 做补偿：

```text
if near_wall and aii_total < 1.5 · max_a:
    is_surface = true
    aii = 1.5 · max_a - (aii_total - aii_fluid)
else if not near_wall and aii_fluid < max_a:
    is_surface = true
    aii = max_a
else:
    aii = aii_fluid
```

这一步的意义是：近壁粒子邻域不完整时，压力矩阵对角项不能太弱，否则投影系统会病态，压力会出现空洞或尖峰。

#### 5.5.6 Step 6：计算右端项 b，也就是 divergence source

右端项 `b` 包含速度散度、边界运动和密度补偿。

对 fluid-fluid 邻居，先计算：

```text
g_ij = -(1 / α_i) · r_ij · |∇W| / ||r_ij||
```

然后加入对称散度项：

```text
b_i += 0.5 · dot(v_i - v_j, g_ij) · ρ0 / Δt
b_j += 0.5 · dot(v_i - v_j, g_ij) · ρ0 / Δt
```

对 ghost 邻居，ghost 表示鱼体表面的运动边界。先取相对速度：

```text
dvel = v_i - v_ghost
```

分解成法向和切向：

```text
n_vel = dot(dvel, n_ghost) · n_ghost
t_vel = dvel - n_vel
```

再构造边界速度：

```text
if dot(dvel, n_ghost) < 0:
    bc_vel = 2 · (n_vel + tangential · t_vel)
else:
    bc_vel = 2 · (separation · n_vel + tangential · t_vel)
```

当前：

```text
VAP_SEPARATION = 0.1
VAP_TANGENTIAL = 0.1
```

加入 ghost 源项：

```text
b_i += ghost_div_scale · dot(bc_vel, g_ij) · ρ0 / Δt
ghost_div_scale = 0.02
```

密度补偿项为：

```text
if ρ_i > ρ0:
    b_i += 0.05 · ρ0 · ((ρ_i - ρ0) / ρ0) / Δt²
```

最后源项整体松弛：

```text
b_i *= VAP_SOURCE_RELAXATION
VAP_SOURCE_RELAXATION = 0.015
```

#### 5.5.7 Step 7：矩阵-向量乘法 A p 与 CG 求解

矩阵 `A` 不显式存储，而是通过矩阵-向量乘法计算：

```text
(A p)_i = aii_i p_i + Σ_j a_ij p_j
```

fluid-fluid 非对角项：

```text
a_ij = -(1 / α_i) · wrr(||x_i - x_j||, h)
```

ghost 不是压力自由度，所以 `A p` 中跳过 ghost 邻居。线性系统为：

```text
A p = b
```

CG 初始化：

```text
p = 0
r = b - A p = b
search = r
```

每轮迭代：

```text
Ap = A search
α_cg = (rᵀr) / (searchᵀAp)
p ← p + α_cg search
r ← r - α_cg Ap
β = (r_newᵀr_new) / (r_oldᵀr_old)
search ← r + β search
```

停止条件：

```text
iteration < VAP_CG_MAX_ITER
sqrt((rᵀr) / n_merged) > VAP_CG_TOL
```

当前：

```text
VAP_CG_MAX_ITER = 50
VAP_CG_TOL = 0.1
```

求得的原始压力会先保存到 debug buffer，然后截断：

```text
p = clamp(p, -VAP_PROJECTION_PRESSURE_ABS_MAX, VAP_PROJECTION_PRESSURE_ABS_MAX)
VAP_PROJECTION_PRESSURE_ABS_MAX = 35000
```

#### 5.5.8 Step 8：用投影压力修正 band 粒子速度

对 fluid 邻居：

```text
dn = -(1 / aii_i) · r_ij · (Δt / ρ0) · |∇W| / ||r_ij||
Δv_i += (p_j - p_i) · dn
```

对 surface 且不靠近 wall 的粒子，当前实现中有一个特殊项：

```text
Δv_i += p_j · dn
```

对 ghost 邻居，不使用 ghost 压力，而是使用速度惩罚：

```text
w = (1 / aii_i) · wrr(||r_ij||, h)
Δv_i += ghost_vel_scale · w · dot(v_ghost - v_i, r_ij) · r_ij
ghost_vel_scale = 0.05
```

为了避免单步投影造成过大速度变化，代码限制：

```text
max_delta_v = particle_spacing · VAP_MAX_PROJECTION_SHIFT_DX / Δt
VAP_MAX_PROJECTION_SHIFT_DX = 0.35
```

最后把 merged 中的 fluid 速度写回原 SPH 粒子数组。

#### 5.5.9 Step 9：压力缩放并写回流体压力

用于速度投影的压力和用于水动力积分的压力不是完全同一个量级。当前做法是先用投影压力修正速度，之后将压力转换为 hydro 输出压力：

```text
p_hydro = clamp(p, -VAP_PRESSURE_ABS_MAX, VAP_PRESSURE_ABS_MAX)
p_hydro *= VAP_PRESSURE_TO_HYDRO
```

当前：

```text
VAP_PRESSURE_ABS_MAX = 500000
VAP_PRESSURE_TO_HYDRO = 0.75
```

然后把 band fluid 的压力写回普通 SPH 粒子压力数组，使后续水压力计算可以从近壁投影压力中取值。

#### 5.5.10 Step 10：由 VAP 压力积分鱼体表面水压力

如果启用 VAP，鱼体表面水压力不是直接从普通 SPH 粒子压力采样，而是从 VAP 近壁粒子压力采样。

力学上仍然是表面积分：

```text
F_face ≈ -p_face n_face A_face
```

再把三角面力分配给三个 FEM 顶点，进入 `boundary_forces_hydro_sub`。每个流体子步都会得到一个水压力贡献，最后在一整个 FEM 步内累加、平均，再传给固体侧。

关键 VAP 参数：


| 参数                                | 当前值     | 含义               |
| --------------------------------- | ------- | ---------------- |
| `VAP_BAND_H_MULT`                 | `2.5`   | band 半径约为 `2.5h` |
| `VAP_CG_MAX_ITER`                 | `50`    | CG 最大迭代          |
| `VAP_CG_TOL`                      | `0.1`   | CG 收敛容差          |
| `MAX_VAP_NEIGHBORS`               | `128`   | 最大 VAP 邻居数       |
| `VAP_PRESSURE_TO_HYDRO`           | `0.75`  | 投影压力转水动力比例       |
| `VAP_PROJECTION_PRESSURE_ABS_MAX` | `35000` | 投影压力截断           |
| ghost divergence scale            | `0.02`  | ghost 散度耦合缩放     |
| ghost velocity scale              | `0.05`  | ghost 速度惩罚缩放     |


## 6. FSI 界面力：从流体压力到 FEM 顶点力

### 6.1 鱼体表面提取

鱼体内部使用四面体网格，流体需要的是表面三角网格。项目从四面体所有面中找只出现一次的面，作为外表面：

```text
一个四面体有 4 个三角面
内部面会被相邻两个四面体共享，出现两次
外表面只出现一次
```

得到的表面三角形用于：

- 给流体提供边界几何。
- 计算三角面法向。
- 将流体力分配回 FEM 顶点。
- 可视化鱼体表面。

### 6.2 压力积分

连续力学中，压力对固体表面的作用为：

```text
dF = -p n dA
```

其中：

- `p` 是流体压力。
- `n` 是表面外法向。
- `dA` 是表面面积元。

离散到三角面上，可以理解为：

```text
F_face ≈ -p_face n_face A_face
```

再将三角面力分配到三个顶点。对于一个三角形，如果最近点或积分点的重心坐标为 `(u, v, w)`，则：

```text
F_i0 += u F_face
F_i1 += v F_face
F_i2 += w F_face
```

这种做法保证了表面力最终进入 FEM 的同一套顶点自由度中。

### 6.3 接触排斥与水压力分离

当前流体力分为两类：

1. contact：粒子靠近或穿透三角面时的弹簧-阻尼排斥力。
2. hydro：由 WCSPH/VAP 压力积分得到的水压力。

工程上分离两者有两个好处：

- 可以单独诊断“几何接触修正”和“真实水动力压力”。
- 可以在调参时关闭接触排斥，只考察水压力反馈。

当前默认设置是关闭 contact repulsion：

```text
contact repulsion = false
```

因此主导鱼体反馈的是 hydro pressure。

### 6.4 显式弱耦合稳定性处理

显式弱耦合容易出现流体力突然过大、固体形变过冲、四面体翻转等问题。当前项目使用几种稳定化手段：

1. 固体高频率：`960 Hz`，减小固体时间步。
2. 流体子步：每个固体步内执行多个 SPH/VAP 子步，默认 `18`。
3. 子步平均：流体力不是取单个瞬时值，而是对子步力平均。
4. 顶点力截断：默认单顶点最大力 `5 N`。
5. 水压力缩放：`hydro force scale = 0.1`。
6. VAP 压力截断：避免投影压力异常尖峰。
7. ghost 耦合缩放：避免表面 ghost 过度主导近壁压力。
8. 体积/翻转诊断：实时检查固体数值稳定性。

## 8. 诊断指标的力学含义

为了判断仿真是否可信，项目不只看动画，还记录一组力学/数值诊断指标。

### 8.1 固体稳定性


| 指标              | 含义                         |
| --------------- | -------------------------- |
| `volume_ratio`  | 当前总体积 / 初始总体积，接近 1 表示体积保持好 |
| `min_tet_ratio` | 最小四面体体积比例，过小说明局部塌缩         |
| `max_tet_ratio` | 最大四面体体积比例，过大说明局部拉伸严重       |
| `inverted_tets` | 翻转四面体数量，非零表示网格拓扑/数值风险      |


这些指标用于判断固体侧是否因为流体力过大或时间步过大而失稳。

### 8.2 流体压力覆盖

表面水压力非零比例用于判断近壁压力是否覆盖到鱼体表面。若大量表面顶点水压力为零，说明 SPH/VAP 近壁采样或投影存在 gap，流体反馈不完整。

### 8.3 COM 与无偏位移

质心位移 `COM displacement` 会受到鱼体整体旋转和姿态影响。无偏位移通过参考构型与当前构型的刚体配准得到，更适合衡量鱼体整体推进，而不是局部摆动造成的表观位移。

### 8.4 力矩

`fsi_force_snapshot` 会计算流体合力关于质心或中心点的力矩，例如绕 x 轴力矩。力矩能反映水压力是否产生合理的滚转/摆动趋势，也能辅助判断压力分布是否偏置。

## 11. 汇报建议结构

推荐按“力学问题 -> 数值方法 -> 实验诊断”组织，而不是按代码目录组织。

1. 问题定义：鱼体肌肉驱动游动是一个移动边界 FSI 问题。
2. 总体方法：显式弱耦合，固体 FEM 与流体 SPH 分开求解，每步交换边界速度和表面力。
3. 固体侧：四面体软体鱼、Corotated FEM、Projective Dynamics、肌肉主动约束。
4. 流体侧：WCSPH 密度/压力/粘性、均匀网格邻域搜索。
5. 近壁增强：VAP band、ghost 边界、压力投影。
6. 力传递：压力积分到三角面，再分配到 FEM 顶点。
7. 稳定性：高频固体步、流体子步、力截断、压力缩放、体积/翻转诊断。
8. 实验展示：可视化鱼体摆动、压力分布、力快照、VAP 粒子追踪。

## 12. 当前工作的亮点与限制

亮点：

- 从力学层面形成了完整的鱼体游动 FSI 链路。
- 固体侧能处理软体大变形和肌肉主动驱动。
- 流体侧用 GPU WCSPH 表示移动边界水体。
- 近壁使用 VAP band 改善压力覆盖和边界条件。
- FSI 力拆分为 contact/hydro/total，便于分析。
- 不只看动画，还输出体积、翻转、压力覆盖、力矩和粒子诊断。

限制：

- 当前是显式弱耦合，不是强耦合；流固之间没有每步迭代到界面平衡。
- 多个物理参数仍是工程调参，未完全由真实鱼体实验标定。
- SPH 分辨率有限，近壁压力仍依赖 VAP 和 ghost 参数。
- 默认关闭接触排斥，主要观察 hydro pressure 贡献。
- 高分辨率流体会显著增加 GPU 计算成本。

后续改进方向：

- 引入半隐式或强耦合 FSI 迭代。
- 用实验数据标定鱼体刚度、肌肉激活和流体参数。
- 优化 ghost 采样，使其与 SPH 粒子分辨率匹配。
- 增加推进效率、尾摆幅值、流场涡结构等评价指标。
- 自动生成课程汇报图表，如压力云图、力矩曲线和无偏位移曲线。

## 13. 总结

当前项目实现的是鱼体游动 FSI 问题的显式弱耦合仿真：固体侧用 SoftCon/Projective Dynamics 处理软体鱼的大变形和肌肉主动收缩，流体侧用 WCSPH 表示水体并用 VAP band 强化近壁压力，界面上通过鱼体表面三角形把流体压力转换为 FEM 顶点外力。它的重点不是单纯展示代码结构，而是展示一个从力学建模、数值离散、界面耦合到稳定性诊断的完整仿真方法。
